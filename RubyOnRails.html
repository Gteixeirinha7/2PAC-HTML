Utilização do Ruby on Rails para se obter o máximo da linguagem! Março 09, 2020

Ruby on Rails surgiu como um framework para a linguagem ruby no ano de 2004, como uma possiblidade de desenvolvimento WEB , e ganhou muita popularidade dentro da comunidade de desenvolvimento mundial. Com tanta popularidade, foi utilizada em diversos projetos grandes, como por exemplo o Github, Airbnb e Basecamp, e utiliza conceitos do MVC e da programação orientada a objetos como base da sua estrutura de desenvolvimento.

Como uma linguagem altamente focada nas boas práticas de codificação e no desenvolvimento ágil, não poderiamos deixar de citar as melhores práticas para tirar o melhor possível do seu código, então vamos lá!!

1. Utilize as Gems!

A lingugagem Rails ficou muita conhecida por conta de gems, uma forma fácil de se importar bibliotecas, e ganhar performance e tempo. Com tantas bibliotecas desenvolvidas por terceiro, é possível encontrar diversas que te auxiliam desde a criar formulários facilmente como a Simple_form como configurar o login de usuários com a Devise ou fazer upload de arquivos com Carrierwave. Não importa qual a sua necessidade, sempre terá uma gem para te auxiliar.


2. Mantenha o seu código limpo!

Através de gems para desenvolvimento como o Rubocop, é possível realizar configurações para que seja possível manter seu código de uma forma limpa e seguindo os padrões impostos pelo seu projeto, através de verificações realizadas pela gem. Com isso temos no fim um projeto entendível para todos que irão entrar no time, além de manter a manutenção e introdução de novas features de forma fácil e ágil.


3. Crie testes automatizados!

E mais uma vez, vamos utilizar as gems! A mais conhecida para realizar testes é a RSpec, responsável por permitir a criação de testes unitários e até mesmo de features completas. Em conjunto com a gem FactoryBot, para criação de dados fakes, é possível navegar por todo projeto fazendo interações, requisições e preenchimentos, tornando o sistema mais confiável e evitando que alterações futuras introduzam bugs e falhas inesperadas, evitando gastos com retrabalhos e desgate com os usuários finais.


Essa é a primeira parte de uma coletanea de artigos que serão lançados semanalmente, sobre melhores práticas e de como tirar o maior poveito possível da sua stack. Continue nos seguindo e acompanhe para fazer parte da nova geração de desenvolvedores!
