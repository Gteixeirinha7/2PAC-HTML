Processos em lotes ou "Batches"

Uso de processos em lotes para grandes volumes de dados Abril 08, 2020

Aqui utilizaremos a linguagem Apex para exemplificar

Como funciona a sintaxe de uma batch:

global class BatchTest implements Database.Batchable<SObject> {

 {0} global Database.QueryLocator start(Database.BatchableContext bc) {
     faz uma seleção de dados no banco (grande volume).
 }

 {1} global void execute(Database.BatchableContext bc, list <P>) {
     faz algum processo com os dados do lote recebido.
 }

 {2} global void finish(Database.BatchableContext bc) {
     faz algum processo após TODOS os lotes serem executados.
 }
}

  • {0} => Executado apenas uma vez, pode selecionar até 50 milhões de registros.
  • {1} => Pode ser executado diversas vezes dependendo da volumetria de dados que foram selecionados.
  • {2} => Executado apenas uma vez, após todos os lotes do passo {1} serem executados.

    Para executar este processo em lotes, o seguinte código deve ser executado:

    Database.executeBatch(new BatchTest(), 5000);

  • "new BatchTest()"

    • É uma instância da classe que contém o processo em lote, é necessário para que o programa entenda a classe que será usada no processo.

  • "5000"

    • Quantidade de registros por lote, por exemplo:

    Se vier um total de 15001 registros, o trecho {1} executará quatro vezes, as três primeiras processando os 15000 primeiros dados e a quarta execução será com apenas o registro que sobrou.
